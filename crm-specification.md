# Rowan Rose Solicitors - CRM Interface Specification

## Document Version: 1.0
## Date: 20 January 2026

---

## Overview

This specification defines a client management interface for an irresponsible lending claims practice. The system must be intuitive for agents, integrate with n8n for workflow automation, and maintain comprehensive audit trails.

---

## Design Principles

| Principle | Implementation |
|-----------|----------------|
| Modern & Professional | Clean lines, generous white space, subtle shadows, professional colour palette |
| Agent Efficiency | Minimal clicks to complete tasks, clear visual hierarchy, keyboard shortcuts |
| Data Integrity | Timestamps on all entries, audit logging, soft deletes only |
| Integration Ready | All actions emit webhooks for n8n consumption |

### Colour Palette Recommendation

| Element | Colour | Usage |
|---------|--------|-------|
| Primary | `#1E3A5F` | Headers, primary buttons |

## Header Components

### Client ID Display
- Position: Top left
- Style: Bordered box, subtle highlight
- Content: Auto-generated unique identifier
- Format: `RR-YYMMDD-XXXX` (e.g., RR-260120-0001)

### Client Name Display
- Position: Centre of header
- Content: `[First Name] [Last Name]`
- Style: Large, bold typography

### Quick Action Buttons
- Position: Top right, 2x2 grid
- Buttons: SMS, Call, Email, WhatsApp
- Behaviour: Opens respective communication modal
- Style: Outlined buttons with icons, fill on hover
- Each action logs to Communication tab and Action Timeline

### Validation Rules
- Sort code: 6 digits, display as XX-XX-XX
- Account number: Exactly 8 digits
- Postcode: Valid UK postcode format
- Date of birth: Must be 18+ years ago

### Action Types to Log

| Category | Actions |
|----------|---------|
| Account | Viewed account, Updated personal details, Updated bank details |
| Claims | Created claim, Updated claim, Changed status, Deleted claim |
| Communication | Email sent/received, SMS sent/received, WhatsApp sent/received, Call logged |
| Documents | Uploaded, Viewed, Deleted, Categorised |
| Notes | Added, Edited, Deleted, Pinned |
| Workflows | Triggered, Step completed, Cancelled, Completed |
| System | Auto-actions, Integrations, Errors |

### Filter Options
- All Actions
- By Agent
- By Action Type
- By Date Range
- By Claim

### Action Log Data Schema

```json
{
  "action_log": {
    "log_id": "string (auto-generated)",
    "client_id": "string (foreign key)",
    "claim_id": "string (foreign key, nullable)",
    "actor_type": "enum: agent | client | system",
    "actor_id": "string (agent_id or 'client' or 'system')",
    "action_type": "string (from action types)",
    "action_category": "string (from categories)",
    "description": "string",
    "metadata": "json (additional context)",
    "timestamp": "timestamp",
    "ip_address": "string (for agents)",
    "user_agent": "string (for agents)"
  }
}
```

## Quick Action Buttons (Header)

### SMS Button
- Opens modal with phone number pre-filled
- Template dropdown for common messages
- Character count display
- Logs to Communication and Action Timeline

### Call Button
- Initiates click-to-call (if VoIP integrated) OR
- Opens modal to log call details post-call
- Fields: Duration, Outcome, Notes
- Logs to Communication and Action Timeline

### Email Button
- Opens email composer modal
- To field pre-filled with client email
- Template dropdown for common emails
- Rich text editor
- Attachment support
- Logs to Communication and Action Timeline

### WhatsApp Button
- Opens WhatsApp Web in new tab with number pre-filled OR
- Opens in-app WhatsApp interface (if integrated)
- Template dropdown for common messages
- Logs to Communication and Action Timeline

## Security Requirements

| Requirement | Implementation |
|-------------|----------------|
| Authentication | JWT tokens, 15-minute expiry, refresh tokens |
| Authorisation | Role-based access control (RBAC) |
| Data Encryption | AES-256 at rest, TLS 1.3 in transit |
| Audit Logging | All actions logged with agent ID, timestamp, IP |
| Session Management | Single session per agent, force logout on new login |
| Password Policy | Minimum 12 chars, complexity requirements, 90-day rotation |
| 2FA | TOTP required for all agents |

### Role Permissions

| Role | Permissions |
|------|-------------|
| Admin | Full access, user management, settings |
| Senior Agent | Full client access, workflow triggers, reports |
| Agent | Client access, limited workflow triggers |
| Viewer | Read-only access |

## Appendix B: API Endpoint Summary

```
# Clients
GET    /api/clients                    List all clients
POST   /api/clients                    Create client
GET    /api/clients/:id                Get client
PUT    /api/clients/:id                Update client
DELETE /api/clients/:id                Soft delete client

# Claims
GET    /api/clients/:id/claims         List client claims
POST   /api/clients/:id/claims         Create claim
GET    /api/claims/:id                 Get claim
PUT    /api/claims/:id                 Update claim
DELETE /api/claims/:id                 Soft delete claim

# Documents
GET    /api/clients/:id/documents      List client documents
GET    /api/claims/:id/documents       List claim documents
POST   /api/documents                  Upload document
GET    /api/documents/:id              Get document
DELETE /api/documents/:id              Soft delete document

# Communications
GET    /api/clients/:id/communications List communications
POST   /api/communications             Create communication

# Notes
GET    /api/clients/:id/notes          List notes
POST   /api/clients/:id/notes          Create note
PUT    /api/notes/:id                  Update note
DELETE /api/notes/:id                  Soft delete note

# Workflows
GET    /api/clients/:id/workflows      List active workflows
POST   /api/workflows/trigger          Trigger workflow
PUT    /api/workflows/:id/cancel       Cancel workflow

# Action Log
GET    /api/clients/:id/actions        List action timeline
GET    /api/claims/:id/actions         List claim actions

# Agents
GET    /api/agents                     List agents
GET    /api/agents/me                  Get current agent
```
